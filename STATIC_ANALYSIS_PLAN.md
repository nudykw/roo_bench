## Критические проблемы (приоритет 1)

### 1. Проблемы с Optional типами (no_implicit_optional)
**Файлы**: `system/ssh_client.py`, `system/gpu_monitor.py`

**Проблема**: PEP 484 запрещает неявные Optional. Параметры с дефолтным значением `None` должны быть явно объявлены как `Optional[T]`.

**Примеры**:
```python
# system/ssh_client.py:11
def __init__(self, host: str = None, user: str = None, key_path: str = None):
    # Должно быть:
    # def __init__(self, host: Optional[str] = None, user: Optional[str] = None, key_path: Optional[str] = None):
```

**Решение**:
- Добавить `from typing import Optional`
- Изменить все параметры с `= None` на `Optional[Type] = None`

**Затронутые файлы**:
- `system/ssh_client.py` (3 параметра)
- Другие файлы с аналогичными проблемами

---

### 2. Несовместимые типы возвращаемых значений
**Файлы**: `system/gpu_monitor.py`

**Проблема**: Функции объявлены с возвращаемым типом `dict`, но возвращают `None`.

**Примеры**:
```python
# system/gpu_monitor.py:83
def get_vram_stats() -> dict:  # Объявлен dict
    ...
    return None  # Возвращает None
```

**Решение**:
- Изменить возвращаемый тип на `Optional[dict[str, Any]]`
- Или изменить логику, чтобы всегда возвращать dict (например, пустой `{}`)

**Затронутые функции**:
- `get_vram_stats()` (строка 95)
- `get_gpu_utilization()` (строка 108)
- Аналогичные функции в других модулях

---

### 3. Операции с None (operator errors)
**Файлы**: `system/gpu_monitor.py`

**Проблема**: Попытки выполнить арифметические операции с типами, которые могут быть `None`.

**Примеры**:
```python
# system/gpu_monitor.py:134
if vram_used / vram_total < 0.1:  # vram_total может быть None
```

**Решение**:
- Добавить проверки на `None` перед операциями
- Использовать guard clauses

**Затронутые строки**:
- 134, 135, 136, 137 (деление и сравнение с None)

---

### 4. Отсутствие аннотаций типов в критических функциях
**Файлы**: `main.py`, `i18n.py`, `ui/curses_selector.py`, тесты

**Проблема**: Функции без аннотаций типов затрудняют статическую проверку и понимание кода.

**Примеры**:
```python
# main.py:801
def parse_args():  # Нет возвращаемого типа
    ...

# i18n.py:312
def get_text(key: str):  # Нет возвращаемого типа
    ...
```

**Решение**:
- Добавить аннотации возвращаемых типов: `-> ReturnType`
- Для функций без возврата: `-> None`
- Для сложных типов использовать `typing` модуль

**Приоритетные файлы**:
- `main.py` (основная логика)
- `i18n.py` (используется везде)
- `ui/curses_selector.py` (UI логика)

---

### 5. Отсутствие аргументов типов для generic типов
**Файлы**: Множество файлов

**Проблема**: Использование `list`, `dict` без указания типов элементов.

**Примеры**:
```python
# system/ssh_client.py:78
def execute_command(...) -> list:  # Должно быть list[str]
    ...

# system/gpu_monitor.py:143
def get_gpu_info() -> list:  # Должно быть list[dict[str, Any]]
    ...
```

**Решение**:
- Заменить `list` на `list[ElementType]`
- Заменить `dict` на `dict[KeyType, ValueType]`
- Использовать `Any` если тип неизвестен

## Некритические проблемы (приоритет 2)

### 6. Длинные строки (E501)
**Количество**: ~489 ошибок

**Проблема**: Строки длиннее 88 символов (настройка в `pyproject.toml`).

**Решение**:
- Разбить длинные строки на несколько
- Использовать скобки для многострочных выражений
- Вынести длинные строки в переменные

**Примеры**:
```python
# До:
if any(kw in combined for kw in ['vision', 'instruct', 'clip', 'image', 'multi modal', 'moar']):

# После:
vision_keywords = ['vision', 'instruct', 'clip', 'image', 'multi modal', 'moar']
if any(kw in combined for kw in vision_keywords):
```

---

### 7. Вызовы функций без типов (no-untyped-call)
**Файлы**: `main.py`, `ui/curses_selector.py`

**Проблема**: Вызовы функций, которые не имеют аннотаций типов.

**Решение**:
- Сначала добавить аннотации в вызываемые функции (см. пункт 4)
- Затем проблема решится автоматически
